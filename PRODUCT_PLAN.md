# 財金資訊系統產品規劃書

## 1. 問題定義

台股投資分析通常需要同時閱讀股價、均線、成交量、月營收、財報、法人買賣超與融資融券等資訊。對財金課程學生、初階投資研究者或需要快速整理個股摘要的人來說，資料來源分散、欄位格式不一致、判讀邏輯不透明，會增加分析門檻。

本系統希望把資料整合、規則判讀與報告輸出放在同一個 Streamlit 儀表板中，讓使用者輸入台股代號後即可取得多面向分析。

## 2. 目標使用者與使用情境

- 財金資訊系統課程學生：展示金融資料 API、資料清理與資訊系統流程。
- 初階投資研究者：快速產生個股初步分析摘要。
- 課堂報告使用者：需要可下載 Markdown 報告，方便整理成作業或簡報。

典型使用情境：

1. 使用者輸入台股代號，例如 2330。
2. 系統自動抓取 yfinance 與 FinMind 資料。
3. 多 Agent 分別提出技術面、基本面、籌碼面與風險面觀點。
4. 系統產生偏多、中立或偏空的規則式結論與 Markdown 報告。

## 3. 資料來源與限制

- yfinance：Open、High、Low、Close、Volume、MA20、MA60、20 日報酬率、平均成交量。
- FinMind：台股基本資料、月營收、財務報表、EPS 或獲利指標、本益比、法人買賣超、融資融券、股利資料。


資料可能延遲或不完整。本系統不宣稱資料完全即時。

## 4. 系統架構

```text
Streamlit UI
  -> Data Agent
      -> yfinance 股價資料
      -> FinMind 基本面與籌碼資料
  -> 技術分析 Agent
  -> 基本面 Agent
  -> 籌碼分析 Agent
  -> 風險控管 Agent
  -> 總結決策 Agent
  -> Markdown 報告輸出
```

FinMind token 由系統管理者於部署環境設定，不由使用者輸入。

讀取順序：

1. Streamlit Secrets：`st.secrets["FINMIND_TOKEN"]`
2. 環境變數：`FINMIND_TOKEN`
3. `.env`
4. 若沒有 token，嘗試公開限制模式

前端只顯示 `FinMind token 已由系統環境設定` 或 `未偵測到 FinMind token，將嘗試公開限制模式`，不顯示 token 內容。

## 5. MVP 功能範圍

- 可輸入台股代號。
- 可按下資料更新按鈕重新抓取資料。
- 顯示最後更新時間。
- 顯示資料來源與資料降級提示。
- 使用 yfinance 計算股價、均線、成交量與報酬率。
- 使用 FinMind 整合月營收、財報、EPS、本益比、法人買賣超、融資融券與股利資料。
- 使用多 Agent 產生辯論式分析。
- 總結決策輸出偏多、中立或偏空。
- 產生可下載 Markdown 報告。
- 報告包含資料來源、資料限制與免責聲明。

## 6. 商業價值與課程展示價值

- 降低多來源金融資料整合門檻。
- 展示 API、資料降級、儀表板、報告輸出的完整資訊系統流程。
- 透過角色型 Agent 強化分析可讀性，讓技術面、基本面、籌碼面與風險面分工清楚。
- 讓使用者可以快速取得研究起點，而不是取代投資判斷。

## 7. 老師評分可能看點

- 是否能實際執行。
- 是否清楚標示 yfinance 與 FinMind 資料來源。
- 是否移除 CSV 作為正式基本面與外資買賣超來源。
- 是否有錯誤處理與降級提示。
- 是否沒有使用 OpenAI API。
- 是否沒有登入與真實交易功能。
- 是否不宣稱資料完全即時。
- 是否每份報告都有免責聲明。
- 是否有明確多 Agent 分工與辯論式輸出。

## 8. 不包含項目

- 不使用 OpenAI API。
- 不建立登入功能。
- 不建立真實交易功能。
- 不讓一般使用者輸入或管理 FinMind token。
- 不提供即時行情保證。
- 不構成任何買賣建議。

## 9. 執行方式

```bash
pip install -r requirements.txt
streamlit run app.py
```

## 10. 免責聲明

本系統分析結果僅供學術研究與投資參考，不構成任何買賣建議。投資人仍應自行評估風險並承擔投資結果。
